    Django PollApp(설문조사 앱) 튜토리얼 - 1) 프로젝트 설치

    1. 가상환경 실행하기 $ myvenv\Scripts\activate

      • 모든 작업은 가상환경(virtualenv) 안에서 하는 것이 좋다. 현재 콘솔 창에서 접두어로 (myvenv)가 안 보인다면 먼저 virtualenv를 활성화해야

    2. 장고가 잘 설치되었는지 확인하기 콘솔 창에

        $ python

    >>> import django
    >>> print(django.get_version())

    3.2.2

    이렇게 버전이 출력되면 잘 설치되었다는 뜻
Ctrl + Z를 눌러 Python을 종료한다.
    1. 장고 기본 골격 만들기
      • 장고의 기본 골격을 만들어주는 스크립트를 실행하는 과정
        $ django-admin startproject mysite

    - 주의 : 장고에서는 디렉토리와 파일명이 매우 중요하다.
장고는 중요한 것들을 찾을 수 있게 특정한 구조를 유지해야 하므로 
파일명을 마음대로 변경하거나 다른 곳으로 옮기면 안 된다.

    위 명령어를 통해 당신의 하드드라이브에는 아래와 같은 구조로 디렉토리와 파일이 생성됐다.

    django
├───manage.py
└───mysite
        settings.py
        urls.py
        wsgi.py
        __init__.py

    각 파일의 역할(한글 튜토리얼 참조):

    • manage.py: Django 프로젝트와 다양한 방법으로 상호작용 하는 커맨드라인의 유틸리티. manage.py 에 대한 자세한 정보는 django-admin and manage.py 명령어로 확인할 수 있다.
    • mysite/ 디렉토리 내부에는 프로젝트를 위한 실제 Python 패키지들이 저장된다. 이 디렉토리 내의 이름을 이용하여, (mysite.urls 와 같은 식으로) 프로젝트의 어디서나 Python 패키지들을 임포트할 수 있다.
    • mysite/init.py: Python으로 하여금 이 디렉토리를 패키지처럼 다루라고 알려주는 용도의 단순한 빈 파일.
    • mysite/settings.py: 현재 Django 프로젝트의 환경 및 구성을 저장한다. Django settings에서 환경 설정이 어떻게 동작하는지 확인할 수 있다.
    • mysite/urls.py: 현재 Django project 의 URL 선언을 저장. Django 로 작성된 사이트의 《목차》 라고 할 수 있다.
    • mysite/asgi.py: ASGI-compatible web server로 내 프로젝트를 구동 하기 위한 파일.
    • mysite/wsgi.py: 현재 프로젝트를 서비스하기 위한 WSGI 호환 웹 서버의 진입점.
